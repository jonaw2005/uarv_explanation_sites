2. Data Flow — Flight Control Commands

sequenceDiagram actor U as 👤 User participant B as buttons.js participant A as controll_api.py participant P as ThreadPoolExecutor participant M as mav_bridge.py participant X as Pixhawk U->>B: Click "ARM" button B->>A: GET /arm_disarm A->>P: executor.submit(bridge.arm_disarm) P->>M: bridge.arm() or bridge.disarm() M->>M: _arm_disarm_sync() — send MAV_CMD_COMPONENT_ARM_DISARM M->>X: MAVLink COMMAND_LONG (param1=1 arm / 0 disarm) X-->>M: COMMAND_ACK (MAV_RESULT_ACCEPTED) M-->>P: return True/False P-->>A: future.result() A-->>B: JSON {status: "arm successful"} B-->>U: Update button status text U->>B: Click "Mode: AUTO" B->>A: POST /change_flightmode {mode: "AUTO"} A->>P: executor.submit(bridge.set_mode, "AUTO") P->>M: bridge.set_mode("AUTO") M->>X: MAVLink SET_MODE (custom_mode = AUTO) X-->>M: (no explicit ack for SET_MODE) M-->>P: return None P-->>A: future.result() A-->>B: JSON {status: "mode_change_requested"} B-->>U: Button text updates U->>B: Click "TAKEOFF" B->>A: GET /takeoff A->>P: executor.submit(bridge.takeoff) P->>M: bridge.takeoff(altitude=50) M->>X: SET_MODE (GUIDED) + MAV_CMD_NAV_TAKEOFF X-->>M: (ack) M-->>P: return None P-->>A: future.result() A-->>B: JSON {status: "takeoff_requested"} B-->>U: Feedback

4. Data Flow — Mission Planning & Upload

sequenceDiagram actor U as 👤 User participant MP as mission_planner.js participant API as controll_api.py participant BR as mav_bridge.py participant PX as Pixhawk %% Mission creation U->>MP: Click map → add waypoint MP->>MP: missionItems.push({type:"waypoint", lat, lon}) U->>MP: Add action (takeoff, loiter, RTL…) MP->>MP: missionItems.push({type:"action", action, param}) %% Upload U->>MP: Click "Upload Mission" MP->>MP: missionToMAVLink(exportMissionToJSON()) Note over MP: Converts JSON → MAVLink items
(seq, frame, command, x, y, z, params) MP->>API: POST /mission_upload [MAVLink items] API->>BR: executor.submit(bridge.upload_mission, items) BR->>PX: MISSION_CLEAR_ALL PX-->>BR: MISSION_ACK (ACCEPTED) BR->>PX: MISSION_COUNT (n items) loop For each item PX-->>BR: MISSION_REQUEST (seq) BR->>PX: MISSION_ITEM_INT (seq, command, x, y, z, params) end PX-->>BR: MISSION_ACK (ACCEPTED) BR-->>API: return True API-->>MP: JSON {status: "upload_complete", items: N} MP-->>U: Alert "Mission uploaded" %% Download U->>MP: Click "Download Mission" MP->>API: GET /mission_download API->>BR: executor.submit(bridge.download_mission) BR->>PX: MISSION_REQUEST_LIST PX-->>BR: MISSION_COUNT (n) loop For each seq 0..n-1 BR->>PX: MISSION_REQUEST_INT (seq) PX-->>BR: MISSION_ITEM_INT end BR-->>API: return [mission items] API-->>MP: JSON [items] MP->>MP: loadMissionFromJSON(items) MP-->>U: Display mission on map
